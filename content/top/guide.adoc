== Foundations ==
This section contains explanations of some of the elements of electronics, the LaunchPad hardware, the Energia software, and the concepts behind them.

Basics::
* http://energia.nu/guide/tutorial_sketch/[Sketch]: The various components of a sketch (Energia program) and how they work.
* http://energia.nu/guide/tutorial_electricity/[Electricity]: The principle that powers electronics.
* http://energia.nu/guide/tutorial_circuits/[Circuits]: The fundamental concept to electrical design.
* http://energia.nu/guide/tutorial_voltage/[Voltage]: Electrical charge transported between two points.
* http://energia.nu/guide/tutorial_current/[Current]: The flow of electrons in a wire.
 

Microcontrollers::
* http://energia.nu/guide/tutorial_digitalpins/[Digital Pins]: How the pins work and what it means for them to be configured as inputs or outputs.
* http://energia.nu/guide/tutorial_analoginputpins/[Analog Input Pins]: Details about the analog-to-digital conversion and other uses of the pins.
* http://energia.nu/guide/tutorial_pwm/[PWM]: How the analogWrite() function simulates an analog output using pulse-width modulation.
* http://energia.nu/guide/tutorial_memory/[Memory]: The various types of memory available on the LaunchPad board.
* Timers: Digital counters that increment or decrement at a fixed frequency used to sync electronic systems.
* Registers: A placeholder for information about some hardware condition.
* http://energia.nu/guide/tutorial_sensors/[Sensors]: Sensors are used to detect and measure analog signals from the environment.
 

Programming Technique::

* http://energia.nu/guide/bit_manipulation/[Bit Manipulation]: How to use bit math.
* http://energia.nu/guide/communication_protocols/[Communication Protocols]: How to use protocols to pass data.

Basic Electronics Components::
* http://energia.nu/guide/breadboard/[Breadboard]: A prototyping surface to create solderless circuits.
* http://energia.nu/guide/wires/[Wires]: Connect electrical components and systems.
* http://energia.nu/guide/resistors/[Resistors]: Alter your circuit voltage and current.
* http://energia.nu/guide/capacitors/[Capacitors]: Store energy in a circuit.
* Filters: Remove or enhance frequency components from the signal.
* Amplifiers: Increase the power of a signal.
* http://energia.nu/guide/transistors/[Transistors]: Amplify and switch electronic signals.
* Inductors: Resist changes in electric current.
* http://energia.nu/guide/switches/[Switches]: Change the path of the electric flow.
* Push Button: Inject a signal into a circuit.
* http://energia.nu/guide/diode/[Diode]: Allows electric flow in only one direction.
* http://energia.nu/guide/leds/[LEDs]: Light emitting diodes act as visual indicators.

== Tutorials ==
Core Functions::
Simple programs that demonstrate basic Energia commands. These are included with the Energia environment; to open them, click the Open button on the toolbar and look in the *examples* folder.  For some examples, additional hardware is required. These can be acquired individually or in popular electronics starter kits.  The http://energia.nu/guide/sidekick/[Sidekick Basic Kit for TI LaunchPad] from Seeedstudio is highly recommended by the Energia community.

. Basics
* http://energia.nu/guide/tutorial_bareminimum/[BareMinimum]: The bare minimum of code needed to start an Energia sketch.
* http://energia.nu/guide/tutorial_blink/[Blink]: Turn an LED on and off.
* http://energia.nu/guide/tutorial_digitalreadserial/[DigitalReadSerial]: Read a switch, print the state out to the Energia Serial Monitor.
* http://energia.nu/guide/tutorial_analogreadserial/[AnalogReadSerial]: Read a potentiometer, print it’s state out to the Energia Serial Monitor.
* http://energia.nu/guide/tutorial_fade/[Fade]: Demonstrates the use of analog output to fade an LED.
* http://energia.nu/guide/tutorial_readanalogvoltage/[ReadAnalogVoltage]: Reads an analog input and prints the voltage to the serial monitor. 

. Digital
* http://energia.nu/guide/tutorial_blinkwithoutdelay/[Blink Without Delay]: Blinking an LED without using the delay() function.
* http://energia.nu/guide/tutorial_button/[Button]: Use a pushbutton to control an LED.
* http://energia.nu/guide/tutorial_debounce/[Debounce]: Read a pushbutton, filtering noise.
* http://energia.nu/guide/tutorial_buttonstatechange/[Button State Change]: Counting the number of button pushes.
* http://energia.nu/guide/tutorial_inputpullupserial/[Input Pullup Serial]: Demonstrates the use of INPUT_PULLUP with pinMode().
* http://energia.nu/guide/tutorial_tone/[Tone]: Play a melody with a Piezo speaker.
* http://energia.nu/guide/tutorial_tone2/[Pitch follower]: Play a pitch on a piezo speaker depending on an analog input.
* http://energia.nu/guide/tutorial_tone3/[Simple Keyboard]: A three-key musical keyboard using force sensors and a piezo speaker.
* http://energia.nu/guide/tutorial_tone4/[Tone4]: Play tones on multiple speakers sequentially using the tone() command.
 

. Analog
* http://energia.nu/guide/tutorial_analoginoutserial/[AnalogInOutSerial]: Read an analog input pin, map the result, and then use that data to dim or brighten an LED.
* http://energia.nu/guide/tutorial_analoginput/[Analog Input]: Use a potentiometer to control the blinking of an LED.
* http://energia.nu/guide/tutorial_analogwrite/[AnalogWrite]: Fade 7 LEDs on and off, one by one, using an MSP430G2 LaunchPad board.
* http://energia.nu/guide/tutorial_calibration/[Calibration]: Define a maximum and minimum for expected analog sensor values.
* http://energia.nu/guide/tutorial_fading/[Fading]: Use an analog output (PWM pin) to fade an LED.
* http://energia.nu/guide/tutorial_smoothing/[Smoothing]: Smooth multiple readings of an analog input.
 

. Communication +
_These examples include code that allows the LaunchPad to talk to Processing sketches running on the computer. For more information or to download Processing, see https://processing.org/[processing.org]._

* http://energia.nu/Tutorial_ReadASCIIString.html[ReadASCIIString]: Parse a comma-separated string of ints to fade an LED.
* http://energia.nu/Tutorial_ASCIITable.html[ASCII Table]: Demonstrates Energia’s advanced serial output functions.
* http://energia.nu/Tutorial_Dimmer.html[Dimmer]: Move the mouse to change the brightness of an LED.
* http://energia.nu/Tutorial_Graph.html[Graph]: Send data to the computer and graph it in Processing.
* http://energia.nu/Tutorial_PhysicalPixel.html[Physical Pixel]: Turn a LED on and off by sending data to your LaunchPad from Processing.
* http://energia.nu/Tutorial_VirtualColorMixer.html[Virtual Color Mixer]: Send multiple variables from LaunchPad to your computer and read them in Processing.
* http://energia.nu/Tutorial_SerialCallResponse.html[Serial Call Response]: Send multiple vairables using a call-and-response (handshaking) method.
* http://energia.nu/Tutorial_SerialCallResponseASCII.html[Serial Call Response ASCII]: Send multiple variables using a call-and-response (handshaking) method, and ASCII-encode the values before sending.
* http://energia.nu/Tutorial_SerialEvent.html[SerialEvent]: Demonstrates the use of SerialEvent().
* http://energia.nu/Tutorial_SwitchCase2.html[Serial input (Switch (case) Statement)]: How to take different actions based on characters received by the serial port.
 
 
. Control Structures

* http://energia.nu/guide/tutorial_ifstatement/[If Statement (Conditional)]: How to use an if statement to change output conditions based on changing input conditions.
* http://energia.nu/guide/tutorial_forloop/[For Loop]: Controlling multiple LEDs with a for loop.
* http://energia.nu/guide/tutorial_whileloop/[Array]: A variation on the For Loop example that demonstrates how to use an array.
* http://energia.nu/Tutorial_WhileLoop.html[While Loop]: How to use a while loop to calibrate a sensor while a button is being read.
* http://energia.nu/guide/tutorial_switchcase/[Switch Case]: How to choose between a discrete number of values. Equivalent to multiple If statements. This example shows how to divide a sensor’s range into a set of four bands and to take four different actions depending on which band the result is in.
* http://energia.nu/guide/tutorial_switchcase2/[Switch Case 2]: A second switch-case example, showing how to take different actions based in characters received in the serial port.

Energia sketches are C/C++ based and compiled with the open-source compiler MSPGCC. The Energia language comes from http://wiring.org.co/[Wiring]. The Energia environment is based on https://processing.org/[Processing] and includes modifications made by Wiring.
